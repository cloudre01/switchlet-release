Switchlet 1.0.0 · Apple Silicon & Intel ·

Default route

http:// & https://

Switchlet takes over from your default browser and asks the menu for a target profile before opening.

What it does

A small app that does one thing well.

01

Per-domain rules

Route *.github.com to Chrome Work, *.youtube.com to Chrome Personal, and the rest to whatever profile you set as default. 

# Rules
*.github.com  → Chrome · Work
*.youtube.com → Chrome · Personal
*             → Chrome · Work

02

Reads existing profiles

No duplicate accounts, no cloud sync, no browser extensions. Switchlet reads your existing Chrome and Firefox profile names from ~/Library/Application Support and shows them in the menu.

03

Tracker stripping

Switchlet strips utm_*, fbclid, gclid, and 30+ other trackers from URLs before they reach the browser. Your links, without the surveillance.

04

Native app handoff

Spotify links open in the Spotify app, Notion in Notion, GitHub in the desktop client. Set per-domain rules to bypass the browser entirely for the apps you live in.

05

Clipboard link picker

Copy multiple links to your clipboard. Trigger Switchlet and it detects every URL. Just pick one and open it with your chosen profile.

How it works

Three steps, then it’s invisible.

  1. 01

    Install

    Download the DMG, drag Switchlet into /Applications. First launch is verified by Gatekeeper. No special permissions, no shell helpers.

  2. 02

    Grant default browser

    Open System Settings → Desktop & Dock → Default web browser and pick Switchlet. macOS confirms and routes http(s):// links to it.

  3. 03

    Set your rules

    Click the menu bar icon, choose a default profile, and add per-domain rules from any page you have open. Switchlet then stays out of your way.

Private by design

No account. No cloud sync. Switchlet reads your browser profile metadata locally, decides which profile should open a link, and hands it off seamlessly.
